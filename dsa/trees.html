Generic Tree

A generic tree is a hierarchical data structure consisting of nodes connected by edges with the following properties:

  • Each node can have zero or more child nodes
  • All nodes except the root have exactly one parent node
  • There are no cycles (a node cannot be its own ancestor)
  • There is exactly one root node (with no parent)

Terminology:

  • Root: The topmost node with no parent
  • Leaf: A node with no children
  • Internal Node: A node with at least one child
  • Degree: Number of children a node has
  • Depth: Number of edges from root to the node
  • Height: Number of edges from node to deepest leaf

Binary Tree

A binary tree is a specialized form of tree where:

  • Each node has at most two children (left and right)
  • The children are distinguished as left child and right child

Types of Binary Trees:

  • Full Binary Tree: Every node has 0 or 2 children
  • Complete Binary Tree: All levels are completely filled except possibly the last level which is filled left to right
  • Perfect Binary Tree: All internal nodes have exactly two children and all leaves are at the same level
  • Degenerate Tree: Each parent node has only one child (essentially a linked list)

Traversals:

  • Inorder (LNR): Left subtree → Node → Right subtree
  • Preorder (NLR): Node → Left subtree → Right subtree
  • Postorder (LRN): Left subtree → Right subtree → Node
  • Level Order: Visit nodes level by level from top to bottom

Binary Search Tree (BST)

A Binary Search Tree is a binary tree with the following properties:

  • For any given node:
    • All values in the left subtree are less than the node's value
    • All values in the right subtree are greater than the node's value
  • No duplicate values are allowed (typically)

Operations:

  • Search: O(h) time complexity (h = tree height)
  • Insertion: Find correct position and add new node
  • Deletion: Three cases:
    • Node is a leaf: Simply remove it
    • Node has one child: Replace with its child
    • Node has two children: Find inorder successor/predecessor, replace, and delete successor/predecessor

Performance:

  • Best case (balanced tree): O(log n) for all operations
  • Worst case (degenerate tree): O(n) (essentially a linked list)

AVL Tree

An AVL Tree (named after inventors Adelson-Velsky and Landis) is a self-balancing binary search tree where:

  • The difference between heights of left and right subtrees (balance factor) is at most 1 for every node
  • This balance condition ensures O(log n) time complexity for all operations

Balance Factor:

For any node, balance factor = height(left subtree) - height(right subtree)

  • Balance factor must be -1, 0, or 1 for the tree to be balanced
  • If balance factor becomes < -1 or > 1, rotations are performed to rebalance the tree

Rebalancing Rules:

  • Left-Left Case (LL): Right rotation
  • Right-Right Case (RR): Left rotation
  • Left-Right Case (LR): Left rotation on left child, then right rotation on node
  • Right-Left Case (RL): Right rotation on right child, then left rotation on node

Advantages:

  • Guaranteed O(log n) time complexity for search, insert, and delete
  • More balanced than standard BST

Disadvantages:

  • More complex implementation due to rotations
  • Slightly higher constant factors due to balance maintenance

Tree Rotations

Rotations are operations performed on binary trees to maintain balance while preserving the BST property.

Left Rotation:

Performed when the right subtree is taller (balance factor < -1):

  1. Make the right child the new root
  2. Move the right child's left subtree to become the original root's right subtree

Right Rotation:

Performed when the left subtree is taller (balance factor > 1):

  1. Make the left child the new root
  2. Move the left child's right subtree to become the original root's left subtree

Rotation Cases in AVL Trees:

  • LL Rotation (Single Right):

    When the left-left subtree is causing imbalance

  • RR Rotation (Single Left):

    When the right-right subtree is causing imbalance

  • LR Rotation (Double: Left then Right):

    When the left-right subtree is causing imbalance

  • RL Rotation (Double: Right then Left):

    When the right-left subtree is causing imbalance

Visualization:

When using the AVL Tree visualization, you'll see rotation operations highlighted in yellow with explanations of which rotation is being performed.

Applications of Trees

Binary Search Trees:

  • Database indexing
  • Filesystem hierarchies
  • Routing tables in networking
  • Implementing ordered maps and sets

AVL Trees:

  • Applications requiring guaranteed O(log n) performance:
    • Database systems
    • Memory allocators
    • 3D graphics (scene graphs)
    • Network routers

Other Tree Applications:

  • B-Trees: Filesystems, databases (handle large amounts of data)
  • Tries: Autocomplete, spell checking, IP routing
  • Heap: Priority queues, scheduling algorithms
  • Syntax Trees: Compiler design, expression evaluation
  • Decision Trees: Machine learning, AI
  • Quad Trees/Octrees: Spatial indexing, collision detection

Why AVL Over BST?

AVL trees are preferred when:

  • You need guaranteed O(log n) performance
  • Search operations are frequent compared to insertions/deletions
  • The dataset is large enough that O(n) worst-case is unacceptable
