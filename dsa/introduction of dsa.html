Comprehensive Notes with Examples, Visuals, and Interactive Animations

What is an Algorithm?

An algorithm is a finite set of precise instructions for performing a computation or solving a problem. It is a well-defined computational procedure that takes some value(s) as input and produces value(s) as output.

  • Sequence of computational steps transforming input to output.
  • Must be clear, unambiguous, and finite.
  • Solves a class of problems, not just a single instance.
Example: Making tea: Add water and milk, boil, add tea leaves, add sugar, serve.

Properties of Algorithms

Property Description
Input Takes zero or more inputs from a specified set.
Output Produces at least one output, related to the input.
Definiteness Every step is clear and unambiguous.
Correctness Produces correct output for every valid input.
Finiteness Terminates after a finite number of steps.
Effectiveness Each step is basic and can be carried out in finite time.
Generality Solves a class of problems, not just a single instance.

Complexity of Algorithms

  • Time Complexity: Amount of time an algorithm takes to run, as a function of input size n.
  • Space Complexity: Amount of memory required by an algorithm as a function of input size.
  • Communication Complexity: (For distributed/parallel algorithms) Measures time taken for communication over a network.
Types of Complexity:
  • Constant Time: O(1)
  • Linear Time: O(n)
  • Logarithmic Time: O(log n)
  • Quadratic Time: O(n¬≤)
  • Exponential Time: O(c‚Åø), c > 1

Analysis of Algorithms

  • Apriori Analysis: Theoretical estimation of resources before actual implementation.
  • Aposteriori Analysis: Empirical analysis after implementation on a specific machine and language.
Why Analyze? Efficient algorithms are necessary for handling large data. The growth of time/space complexity with input size is key for comparing algorithms.
